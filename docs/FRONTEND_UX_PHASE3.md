# Frontend UX Specification - EHS Portal Phase 3: Multi-Organisation & Enterprise Reporting

## 1. Overview

This document defines the user interface specifications for Phase 3 of the EHS Portal. Phase 3 introduces organisation settings, user management, and data export capabilities.

**New Screens:**
- Organisation Settings Page (Admin)
- User Management Page (Admin)
- Reports/Exports Page (Manager, Admin)

**Updated Components:**
- Header (organisation branding)
- Dashboard (threshold-based KPI coloring)
- List pages (export buttons)

### 3.3 States

**Loading State:**
- Show skeleton loaders for all sections
- Disable all inputs

**Error State (load failure):**
- Show error alert: "Failed to load organisation settings"
- Show "Retry" button

**Save States:**
- Profile section: Independent save button, shows spinner during save
- Logo section: Progress bar during upload, then success/error toast
- Thresholds section: Independent save button

## 7. Dashboard Updates

### 7.1 KPI Card Coloring

Dashboard KPI cards now use organisation threshold settings for coloring.

**Color Logic:**
```javascript
const getKpiColor = (value, warningThreshold, criticalThreshold) => {
  if (value >= criticalThreshold) return 'red';    // Critical
  if (value >= warningThreshold) return 'yellow';  // Warning
  return 'green';                                   // Normal
};
```

**Example:**
- Open Incidents: 8
- Warning threshold: 5
- Critical threshold: 10
- Result: Yellow (warning) background
