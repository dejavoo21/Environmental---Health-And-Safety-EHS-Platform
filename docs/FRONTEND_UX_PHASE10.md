## 1. Overview

Phase 10 introduces the **Integrations** admin page for managing SSO, API clients, and webhooks. This document specifies the UX design for all Phase 10 frontend components.

### 1.1 Design Principles

- **Wizard-style Configuration:** Guide admins through complex SSO setup
- **Security-first Display:** Mask secrets, show-once patterns for keys
- **Clear Status Indicators:** Health status for all integrations
- **Actionable Feedback:** Error messages with resolution guidance

## 11. Responsive Design

### 11.1 Mobile Considerations

- Integrations page accessible but optimised for desktop admin use
- Tables collapse to card view on mobile
- Modals become full-screen on mobile
- Copy buttons work with mobile clipboard

### 11.2 Breakpoints

| Breakpoint | Behaviour |
|------------|-----------|
| < 768px | Single column, cards instead of tables |
| 768-1024px | Two column layout for forms |
| > 1024px | Full layout with sidebar detail panels |
